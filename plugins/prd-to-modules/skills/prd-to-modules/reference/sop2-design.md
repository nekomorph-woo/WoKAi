# SOP2: 模块设计

## 执行步骤

1. **准备上下文包**：为每个模块准备设计所需的全部信息
   - PRD 中与该模块相关的需求
   - _registry.md 中该模块的依赖/被依赖关系
   - SOP1 中收集的模块间接口契约
2. **并行设计**（最多 3 个 subagent 同时工作）
   - 将模块列表分组，每组最多 3 个
   - 每组并行 spawn general-purpose subagent
   - 完成一组后再处理下一组
3. **遵循 design-an-interface 哲学**
   - 为每个模块的对外接口考虑多种设计方案
   - 从简洁性、通用性、深度、误用难度等维度评估
   - 选择最优方案或综合多个方案
4. **输出 design.md**

## subagent 提示模板

```
为以下模块设计完整的对内对外接口和架构：

## 模块信息
- 名称: <name>
- 职责: <从 registry 获取>
- 依赖的模块: <列表>
- 被依赖的模块: <列表>

## PRD 需求
<与该模块相关的 PRD 片段>

## 依赖分析
<该模块依赖的接口/模型/方法>
<该模块需要提供的接口/模型/方法>

## 设计要求
1. 遵循 design-an-interface 哲学（Design It Twice）
2. 模块间接口按 API 协议设计：仅通过入参/出参通信，杜绝接口调用产生副作用（写操作/消息传递除外），最大化降低耦合度
3. 设计对内对外接口、数据模型、核心方法
4. 标注私有（internal）和公共（public）
5. 确保模块自闭环——design.md 足以独立指导实现
6. 使用模板格式输出到 plans/modules/<name>/design.md
```

设计原则和低耦合要求已在 design.md 模板中固化，subagent 遵循模板即可。

## 模块间接口低耦合原则

模块间接口按 API 协议设计：
- 仅通过入参/出参通信，杜绝接口调用产生副作用（写操作/消息传递除外）
- 最大化降低调用者与被调用者的耦合度
- VO/DTO/DO 等耦合模型在 SOP3 中自动提取为公共产物

# SOP1: 模块识别与依赖分析

## 执行步骤

1. **读取 PRD**：从上下文或文件获取完整 PRD
2. **识别模块**：分析 PRD 中的功能点，按纵向贯穿原则拆分
   - 每个模块应覆盖完整的业务/功能闭环
   - 模块可以是技术模块（如认证、存储）或业务模块（如用户管理、订单处理）
   - 单个模块过小时考虑合并，过大时考虑拆分
3. **分析依赖关系**：对每对模块，识别
   - A 依赖 B 的什么能力/接口/模型/方法
   - B 能够提供什么
   - 依赖方向（单向/双向）
4. **输出 _registry.md**

## subagent 使用

使用 1 个 Explore subagent：

```
读取 PRD [文件路径或内容]，执行模块拆分分析：
1. 识别所有需要实现的功能/技术模块
2. 分析模块间依赖关系
3. 输出结构化的模块注册表

模块应纵向贯穿（覆盖完整功能闭环），而非按技术层水平拆分。
```

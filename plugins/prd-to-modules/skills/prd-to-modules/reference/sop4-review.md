# SOP4: 审查与修正

## 执行步骤

1. **职责交叉检查**：
   - 同一功能是否出现在多个模块中
   - 模块边界是否清晰
   - 是否存在循环依赖
2. **自闭环验证**：
   - 每个模块的依赖是否都有提供方
   - 每个模块的公共接口是否被其他模块需要
   - 模块是否可以独立实现和测试
3. **修正操作**：
   - 职责交叉 → 明确归属或拆分出新模块
   - 循环依赖 → 引入中间模块或重构接口
   - 缺失依赖 → 补充到 registry 或调整设计
4. **更新输出**：同步更新 _registry.md 和受影响的 design.md

## 检查清单

- [ ] 无循环依赖
- [ ] 无职责交叉
- [ ] 每个模块依赖可满足
- [ ] 每个模块可独立实现
- [ ] 公共产物已正确引用

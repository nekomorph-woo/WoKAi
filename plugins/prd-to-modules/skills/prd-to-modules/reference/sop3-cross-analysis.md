# SOP3: 交叉分析与公共提取

## 执行步骤

1. **读取所有 design.md**
2. **交叉对比**：
   - 识别多个模块中重复定义的模型/类型
   - 识别相似或相同的方法签名
   - 识别可抽取为公共模块的功能
3. **提取公共产物**：
   - 创建公共模型定义（如 `plans/modules/_shared/models.md`）
   - 创建公共模块设计（如 `plans/modules/_shared/common.md`）
4. **更新模块设计**：
   - 各模块 design.md 中引用公共产物
   - 移除重复定义
   - 标注公共依赖关系

## 公共产物判断标准

| 场景 | 处理方式 |
|------|----------|
| 2+ 个模块使用相同数据模型 | 提取到公共模型文件 |
| 2+ 个模块有相似的辅助方法 | 考虑公共工具模块 |
| 单独模块使用的功能 | 保留在模块内部 |
| 概念上相似但细节不同 | 不强制合并，标注差异 |

## 完成后确认

向用户展示提取结果摘要：

```
📋 公共产物提取结果
├── 新增公共模块: <列表或"无">
├── 公共模型: <列表>
└── 公共方法: <列表>

受影响的模块: <因公共提取而修改 design.md 的模块列表>
```

用户确认后再进入 SOP4 审查。
